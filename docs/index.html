What this product is

Digibility SEO Intelligence is a paid add-on module for Digibility’s visibility platform. It runs as an independent Vite + React + TypeScript app during development, then plugs into Digibility Core later — reusing auth, design system, and platform principles without rebuilding them.

Positioning: an SEO execution cockpit (audit → recommendations → approval → content → performance → diagnosis → off-page → AI visibility), not a keyword-research clone.

Architecture in one picture

Browser (React) → Service adapter → Supabase PostgREST + RLS → Postgres tables / RPCs
Crawler worker ⇄ service_role RPCs → Crawl tables → publish → Audit / Page Inventory
Ownership worker ⇄ DNS-TXT claim/result RPCs → seo_ownership_verifications

Locked design decisions

DecisionMeaning
No customer BFF Frontend talks to Supabase directly with the anon key. Authorization is Postgres RLS + guarded SECURITY DEFINER RPCs.
Website is the core entity Every operational SEO record carries website_id + denormalized website_url.
Mock / Supabase dual mode VITE_SEO_DATA_MODE defaults to mock; Supabase mode falls back to mock on config/errors (writes can opt out of masking).
SEO roles ≠ platform roles SEO roles live on seo_workspace_members.seo_role: owner / admin / team_member / client.
Workers are trusted servers Service-role keys exist only in crawler-worker/, never in the browser.
