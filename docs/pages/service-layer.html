Adapter pattern

Service layer

The abstraction boundary lives inside the frontend — not as a network BFF. Every wired feature goes through runWithServiceAdapter.

Data mode switch

ModeWhenBehavior
mock (default) Missing/invalid env or explicit mock Deterministic in-memory / mock modules; zero Supabase writes
supabase VITE_SEO_DATA_MODE=supabase + valid URL/anon key Real PostgREST/RPC calls with user JWT 
// Conceptual
runWithServiceAdapter({
  label: "auditService.fetchLatest",
  mock: () => mockFetch…(),
  supabase: () => supabaseFetch…(),
  fallbackToMockOnError: true,   // default for reads
  // writes often set false so real errors surface
})

Write pattern (guarded RPCs)

Status machines never do update({ status }) from the client. They call a SECURITY DEFINER RPC and use a non-masking write helper so backend denials are shown, not silently replaced by mock success.

UI action → public *Service → run*Write helper → seo*SupabaseService.rpc(…)
