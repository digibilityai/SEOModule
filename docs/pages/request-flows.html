1. Customer login & route gate

A
Anonymous hits /seo/audit (Supabase mode)

ProtectedRoute finds no session → redirect to /seo/login?returnTo=/seo/audit. No product content rendered.

↓
B
Sign-in on chromeless login page

Supabase Auth email/password (existing Digibility users). Session JWT stored by supabase-js.

↓
C
useSeoAccess() checklist

1) session · 2) has_seo_module_access · 3) workspace membership · 4) active website (unless allowSetup). Admin preview also requires seo_is_global_admin.

↓
D
Deep-link restored

User lands on original returnTo. Sign-out clears user-scoped React Query cache + active website.

Mock mode bypasses this gate

2. Typical authenticated read

Page
  → hook (useQuery)
    → *Service.fetch…()
      → runWithServiceAdapter({ mock, supabase })
        → seo*SupabaseService
          → supabase.from('seo_…').select(…).eq('website_id', …)
            → RLS: workspace member only
              → map rows → UI types

Example tables: seo_audit_runs, seo_page_inventory, seo_authority_opportunities.

7. Crawl request / cancel / poll

1
UI: /seo/audit → Start crawl

Owner/admin/team_member (client read-only). Two-step confirm.

↓
2
Orchestration RPC

seo_crawl_request_audit atomically creates a linked seo_audit_runs row + seo_crawl_jobs row and returns both ids (never “latest run” guessing).

↓
3
UI polls Supabase only

useWebsiteCrawl reads job status every ~4s while active; pauses when tab hidden; stops at terminal.

Supabase = single status source
↓
4
Cancel (optional)

seo_crawl_cancel → cancellation_requested → worker ack → cancelled.

UI: OwnershipVerificationPanel on /seo/websites. No status RPC — absence of a row means unverified. DNS resolution is not done in these customer RPCs; the worker does that.

9. Worker job claim (service role)

crawler-worker (service_role)
  → rpc('seo_crawl_claim_job')          -- FOR UPDATE SKIP LOCKED + lease_token
  → discovery / extraction / issues
  → rpc('seo_crawl_worker_record_*')
  → rpc('seo_crawl_worker_publish_results')
  → rpc('seo_crawl_worker_complete' | partial | fail | …)

verify-once mode:
  → rpc('seo_ownership_verification_claim')
  → DNS TXT resolve
  → rpc('seo_ownership_verification_record_result')
Authenticated/anon EXECUTE is revoked on these worker RPCs. If a browser somehow called them, PostgREST would deny.
