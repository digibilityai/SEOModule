Auth & access

No separate SEO identity system. MVP uses standalone Supabase Auth (same project / users as Digibility) with a future parent-identity adapter seam.

Decision (ADR Option C hybrid)

  • Now: customer login via Supabase Auth at /seo/login
  • Later: parent Digibility SSO adapter (not required for MVP)
  • Always: RLS + guarded RPCs remain authoritative; UI gates are UX only

SEO roles vs platform roles

SEO role (membership)Typical powers
ownerFull SEO + billing/team (platform-dependent)
adminManage SEO work; approve high-risk; ownership mutations
team_memberRun audits/crawls, create campaigns, many transitions; not approve campaigns
clientView reports/results; limited low-risk actions; often read-only for writes

Platform profiles.role (super_admin / admin / …) is separate. Global SEO admin checks use seo_is_global_admin for admin-preview.

Session hygiene

  • SessionSync prevents cross-user cache leakage
  • Sign-out clears user-scoped React Query keys + active website selection
  • Deep links restore via safe internal returnTo
  • Mock mode bypasses route protection for local preview
